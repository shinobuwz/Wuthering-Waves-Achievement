## 新增需求

### 需求:OCR识别成就名称
系统必须对每条成就条目的名称区域进行 OCR 识别，提取成就名称文字。

#### 场景:识别已完成成就名称
- **当** 成就条目图像中包含成就名称（如"兮历八方·今州"）
- **那么** 系统必须使用 onnxocr 引擎识别出成就名称文字，返回识别结果字符串

#### 场景:识别包含特殊字符的名称
- **当** 成就名称包含特殊字符（如引号「」、感叹号！、中间点·）
- **那么** 系统必须正确识别或在模糊匹配阶段容忍这些字符的识别偏差

#### 场景:中点字符变体归一化
- **当** OCR 将间隔号·(U+00B7)误识别为日文中点・(U+30FB)或其他变体（•∙･）
- **那么** 系统必须在匹配前将所有中点变体统一替换为·(U+00B7)后再计算编辑距离

### 需求:成就名称模糊匹配
系统必须将 OCR 识别的成就名称与 base_achievements.json 中的成就进行模糊匹配，找到最佳匹配项。

#### 场景:精确匹配
- **当** OCR 识别结果与数据库中某条成就名称完全一致
- **那么** 系统必须立即返回该成就的编号，匹配置信度为 1.0，跳过后续匹配计算

#### 场景:模糊匹配成功
- **当** 归一化后的 OCR 结果与数据库名称的编辑距离（Levenshtein distance）<= 名称长度的 40%
- **那么** 系统必须返回编辑距离最小的成就编号和匹配置信度

#### 场景:匹配失败
- **当** OCR 识别结果与所有成就名称的编辑距离均 > 名称长度的 40%
- **那么** 系统必须将该条目标记为"未匹配"，记录 OCR 原文以供用户审查

### 需求:判断成就完成状态
系统必须判断每条成就是否已完成。

#### 场景:已完成成就
- **当** 成就条目右侧区域的 OCR 识别结果匹配日期格式（YYYY/MM/DD）
- **那么** 系统必须将该成就标记为"已完成"

#### 场景:未完成成就
- **当** 成就条目右侧区域的 OCR 识别结果为"进行中"
- **那么** 系统必须将该成就标记为"未完成"

#### 场景:无法判断状态
- **当** 右侧区域 OCR 结果既不匹配日期格式也不匹配"进行中"
- **那么** 系统必须将该条目标记为"状态不确定"，保留原有状态不修改

### 需求:图像预处理
系统必须对成就条目图像进行预处理以提高 OCR 识别率。

#### 场景:预处理流程
- **当** 截取到成就条目的原始图像
- **那么** 系统必须进行对比度增强和/或二值化处理，再送入 OCR 引擎
