## 上下文

当前 OCR 扫描功能只有全局扫描模式（`scan_all_tabs()`），自动遍历所有一级/二级 Tab。用户无法只扫描当前可见页面，在调试坐标参数或验证单个分类时效率低下。

现有代码结构：
- `core/achievement_ocr.py`：已有 `scan_with_scroll()` 函数可扫描单个二级 Tab 的完整列表，`scan_all_tabs()` 是遍历所有 Tab 的入口
- `core/ocr_tab.py`：`OCRScanWorker` 线程直接调用 `scan_all_tabs()`，无模式分支

## 目标 / 非目标

**目标：**
- 让用户可以选择「单页扫描」或「全局扫描」模式
- 单页扫描：扫描当前游戏内可见的成就列表（含自动滚动到底部），不做 Tab 切换
- 全局扫描：保持现有行为不变
- 多次单页扫描的结果累积合并，不丢失之前的扫描结果

**非目标：**
- 不修改 OCR 识别管线（模板匹配、文字识别、模糊匹配）
- 不修改滚动参数和像素坐标
- 不增加自动选择性扫描（如"只扫未完成的"）

## 决策

### 决策 1：模式选择 UI 使用 QComboBox

在扫描控制区域的"开始扫描"按钮左侧放置一个 QComboBox，选项为"单页扫描"和"全局扫描"（默认）。

**替代方案：**
- QRadioButton 组：占用更多水平空间，与现有紧凑布局不协调
- 两个独立按钮（"扫描当前页"/"扫描全部"）：增加了按钮数量，且需要额外的状态管理

**理由：** QComboBox 最节省空间，语义清晰，且在扫描过程中禁用即可防止误切换。

### 决策 2：单页扫描复用 scan_with_scroll

单页扫描直接调用现有的 `scan_with_scroll(hwnd, ocr_model, db, ...)` 函数，该函数已能扫描当前可见 Tab 的完整列表（含滚动到底部和去重）。不需要创建新的扫描函数。

**替代方案：**
- 只调用 `scan_single_page()` 不滚动：用户只能看到一屏的成就，不实用
- 新建一个 `scan_current_tab()` 包装函数：不必要的抽象层

**理由：** `scan_with_scroll` 已经是扫描单个 Tab 的完整实现，直接复用最简单。

### 决策 3：结果累积策略

单页模式下，每次扫描结果与之前的结果合并（基于成就 ID 去重，新结果覆盖旧结果同 ID 条目）。全局扫描模式每次清空重新开始。

**替代方案：**
- 所有模式都累积：全局扫描会产生混乱的残留数据
- 所有模式都清空：单页扫描丢失之前分类的结果

**理由：** 单页模式的使用场景就是分批扫描不同分类然后合并；全局模式本身就是完整扫描，不需要累积。

## 风险 / 权衡

- [单页扫描无 Tab 信息] → 进度显示无法标注"当前一级/二级分类"，改为显示"当前页面"或仅显示扫描统计
- [用户误用单页模式扫描同一页多次] → 基于成就 ID 去重，重复扫描不会产生重复条目
- [模式切换时机] → 扫描进行中禁用模式切换 ComboBox，避免中途切换导致行为不一致
