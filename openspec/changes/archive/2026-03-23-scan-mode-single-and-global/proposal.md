## 为什么

当前 OCR 成就扫描只有一种模式：全自动遍历所有一级/二级 Tab 的全局扫描。用户在调试像素坐标、验证单个分类的识别效果时，必须等待全部 Tab 扫描完成，效率很低。增加单页扫描模式让用户可以手动切换到目标 Tab 后只扫描当前页面，大幅提升调试和日常使用的灵活性。

## 变更内容

- **新增**：单页扫描模式 —— 用户手动切换游戏内 Tab，点击扫描后仅扫描当前可见页面（含自动滚动到底部）
- **新增**：扫描模式选择 UI —— 在扫描控制区域增加模式切换（单页扫描 / 全局扫描）
- **修改**：将现有的全自动扫描重命名为「全局扫描」模式，行为不变
- **修改**：结果累积策略 —— 单页模式下多次扫描的结果应累积合并，而非覆盖

## 功能 (Capabilities)

### 新增功能

- `scan-mode-selection`: 扫描模式选择与切换机制，包括 UI 控件和模式状态管理
- `single-page-scan`: 单页扫描模式的核心行为 —— 仅扫描当前可见 Tab 页面，支持滚动到底部，结果累积合并

### 修改功能

- `ocr-scan-tab`: 扫描控制区域增加模式选择控件，进度显示需适配两种模式

## 影响

- `core/ocr_tab.py`: UI 层增加模式选择控件，OCRScanWorker 需根据模式分发不同扫描逻辑
- `core/achievement_ocr.py`: 暴露单页扫描入口（`scan_with_scroll` 已存在，需封装为独立可调用接口）
- `openspec/specs/ocr-scan-tab/spec.md`: 需增加模式选择相关的规范条目
