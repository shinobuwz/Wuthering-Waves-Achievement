## 新增需求

### 需求:检测鸣潮游戏窗口
系统必须通过 Win32 API (ctypes) 自动检测当前正在运行的鸣潮游戏窗口，获取窗口句柄、位置和大小信息。

#### 场景:游戏窗口存在且在前台
- **当** 鸣潮游戏窗口正在运行且处于前台
- **那么** 系统必须返回窗口句柄 (HWND)、窗口左上角坐标 (x, y) 和窗口大小 (width, height)

#### 场景:游戏窗口不存在
- **当** 未检测到鸣潮游戏窗口
- **那么** 系统必须返回错误信息，提示用户先启动游戏

#### 场景:窗口标题匹配
- **当** 系统搜索游戏窗口时
- **那么** 系统必须通过窗口标题关键词匹配（支持 "Wuthering Waves" 和中文标题），如果 FindWindow 精确匹配失败，必须回退到枚举窗口进行模糊匹配

### 需求:前台窗口截图
系统必须对检测到的游戏窗口进行前台截图，生成完整的游戏界面图像。

#### 场景:成功截图
- **当** 游戏窗口已检测到且分辨率为 1920x1080
- **那么** 系统必须使用 mss 库截取窗口所在区域的屏幕图像，返回 numpy array (BGR 格式)

#### 场景:分辨率不匹配
- **当** 检测到的游戏窗口分辨率不是 1920x1080
- **那么** 系统必须提示用户将游戏设置为 1920x1080 分辨率

### 需求:成就区域裁剪
系统必须从完整截图中裁剪出成就列表的各个关键区域。

#### 场景:裁剪右侧成就列表
- **当** 获取到 1920x1080 的游戏截图
- **那么** 系统必须按预定义的像素坐标裁剪出右侧成就条目列表区域（约 x:530 到 x:1920 范围）

#### 场景:裁剪单条成就
- **当** 获取到右侧成就列表区域图像
- **那么** 系统必须按固定高度将列表分割为独立的成就条目图像
