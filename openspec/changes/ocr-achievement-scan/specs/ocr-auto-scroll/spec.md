## 新增需求

### 需求:模拟鼠标滚轮翻页
系统必须使用 pyautogui 模拟鼠标滚轮操作，使右侧成就列表向下滚动以显示更多成就条目。

#### 场景:激活窗口
- **当** 需要执行滚动操作
- **那么** 系统必须先将鼠标移动到屏幕中心并点击，确保游戏窗口处于激活状态

#### 场景:执行滚动
- **当** 当前页面的成就条目已扫描完成且未到达底部
- **那么** 系统必须连续发送多次 `pyautogui.scroll()` 事件（默认 scroll(-160) x 15 次），利用惯性叠加实现所需的滚动幅度

#### 场景:等待滚动动画
- **当** 滚轮操作已执行
- **那么** 系统必须等待固定时间（默认 0.8 秒）让滚动动画完成后再进行下一次截图

### 需求:滚动到底检测
系统必须检测成就列表是否已滚动到底部，自动停止滚动。

#### 场景:检测到底部
- **当** 新一轮截图 OCR 识别出的成就名称集合与上一轮完全重复（无新成就出现）
- **那么** 系统必须判定已到达底部，停止滚动循环

#### 场景:存在新成就
- **当** 新一轮截图中出现了之前未识别过的成就名称
- **那么** 系统必须继续滚动扫描

### 需求:一级Tab切换
系统必须能够点击左侧图标栏切换一级分类Tab。

#### 场景:点击一级Tab图标
- **当** 需要切换到指定一级分类（索拉漫行/铿锵刃鸣/长路留迹/诸音声轨）
- **那么** 系统必须根据预设百分比坐标（x=0.0417，y依次为0.1778/0.2981/0.4343/0.5537）换算为窗口像素坐标并点击，等待0.8秒让页面切换动画完成

### 需求:二级Tab切换
系统必须能够识别并点击左侧列表中的二级分类Tab。

#### 场景:识别可见二级Tab
- **当** 需要切换二级分类
- **那么** 系统必须对二级Tab列表区域（x:0.1005~0.3479，y:0.1796~1.0）整体做一次OCR，从 detect box 提取各Tab的y中心坐标，用编辑距离匹配到已知Tab列表，返回[(名称, center_y_pct, ocr原文)]

#### 场景:点击二级Tab
- **当** 已识别到目标二级Tab的y坐标
- **那么** 系统必须点击对应百分比坐标，等待0.5秒

#### 场景:二级Tab列表需要滚动
- **当** 目标二级Tab不在当前可见区域内（该分类Tab数量超过一屏）
- **那么** 系统必须在二级Tab区域连续发送 scroll(-160) x 16 次后重新识别

### 需求:扫描去重
系统必须对多次截图中重复出现的成就条目进行去重，确保每条成就只记录一次。

#### 场景:滚动后部分重叠
- **当** 滚动后部分成就条目与上一次截图重叠（同一条成就出现两次）
- **那么** 系统必须基于匹配到的成就编号进行去重，只保留首次识别的结果

#### 场景:全量去重
- **当** 整个子分类扫描完成
- **那么** 系统必须返回去重后的完整成就列表及其完成状态
