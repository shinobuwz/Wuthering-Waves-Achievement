## 新增需求

### 需求:模拟鼠标滚轮翻页
系统必须模拟鼠标滚轮操作，使右侧成就列表向下滚动以显示更多成就条目。

#### 场景:执行滚动
- **当** 当前页面的成就条目已扫描完成且未到达底部
- **那么** 系统必须将鼠标移动到右侧成就列表区域内，模拟鼠标滚轮向下滚动固定距离（滚动幅度对应 N 条成就高度）

#### 场景:等待滚动动画
- **当** 滚轮操作已执行
- **那么** 系统必须等待固定时间（默认 0.5~1 秒）让滚动动画完成后再进行下一次截图

### 需求:滚动到底检测
系统必须检测成就列表是否已滚动到底部，自动停止滚动。

#### 场景:检测到底部
- **当** 新一轮截图 OCR 识别出的成就名称集合与上一轮完全重复（无新成就出现）
- **那么** 系统必须判定已到达底部，停止滚动循环

#### 场景:存在新成就
- **当** 新一轮截图中出现了之前未识别过的成就名称
- **那么** 系统必须继续滚动扫描

### 需求:扫描去重
系统必须对多次截图中重复出现的成就条目进行去重，确保每条成就只记录一次。

#### 场景:滚动后部分重叠
- **当** 滚动后部分成就条目与上一次截图重叠（同一条成就出现两次）
- **那么** 系统必须基于匹配到的成就编号进行去重，只保留首次识别的结果

#### 场景:全量去重
- **当** 整个子分类扫描完成
- **那么** 系统必须返回去重后的完整成就列表及其完成状态
