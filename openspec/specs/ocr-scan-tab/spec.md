# ocr-scan-tab

## 目的

在应用主窗口中提供"OCR 扫描"标签页，为用户提供游戏窗口检测、扫描控制、实时进度显示、结果预览及将扫描结果写入用户进度等完整交互界面。

## 需求

### 需求:OCR扫描Tab页面
系统必须在现有应用中新增一个"OCR 扫描"标签页，与成就管理、统计图表、数据爬取并列。

#### 场景:Tab页注册
- **当** 应用启动时
- **那么** 系统必须在 main_window.py 的 tab_widget 中注册 OCR 扫描 Tab，显示标题"OCR 扫描"

#### 场景:主题适配
- **当** 用户切换明/暗主题
- **那么** OCR 扫描 Tab 必须跟随主题切换样式

### 需求:窗口检测控制
Tab 页必须提供游戏窗口检测的控制按钮和状态显示。

#### 场景:点击检测按钮
- **当** 用户点击"检测游戏窗口"按钮
- **那么** 系统必须调用 game_capture 模块检测鸣潮窗口，成功时显示"已检测到窗口"及窗口信息，失败时显示错误提示

### 需求:扫描控制
Tab 页必须提供开始和停止扫描的控制。

#### 场景:开始扫描
- **当** 游戏窗口已检测到，用户点击"开始扫描"按钮
- **那么** 系统必须启动 OCR 扫描流程（截图 → OCR → 滚动 → 循环），按钮变为"停止扫描"

#### 场景:停止扫描
- **当** 扫描正在进行中，用户点击"停止扫描"按钮
- **那么** 系统必须在当前轮次完成后安全停止扫描

#### 场景:未检测窗口时点击扫描
- **当** 游戏窗口未检测到，用户点击"开始扫描"
- **那么** 系统必须提示用户先检测游戏窗口

### 需求:实时进度显示
Tab 页必须在扫描过程中实时显示进度信息。

#### 场景:显示扫描进度
- **当** 扫描正在进行中
- **那么** 系统必须实时显示：已识别成就数量、当前轮次、扫描状态文字

### 需求:扫描结果预览
Tab 页必须显示扫描结果的预览列表。

#### 场景:展示识别结果
- **当** 扫描完成或扫描过程中
- **那么** 系统必须在表格或列表中展示每条已识别成就的名称、匹配结果、完成状态

#### 场景:标记未匹配项
- **当** 某条成就 OCR 结果未能匹配到数据库
- **那么** 系统必须在结果列表中以醒目方式标记该条目

### 需求:结果写入用户进度
扫描结果必须能写入现有的用户进度数据。

#### 场景:同步到用户进度
- **当** 用户确认扫描结果（或扫描完成后自动同步）
- **那么** 系统必须通过 config.save_user_progress() 将识别到的成就状态更新到当前用户的 user_progress 文件中

#### 场景:不覆盖已有状态为更低状态
- **当** 某条成就在用户进度中已标记为"已完成"，但 OCR 扫描结果为"未完成"
- **那么** 系统禁止将状态降级，必须保留"已完成"状态
