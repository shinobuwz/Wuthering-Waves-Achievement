# single-page-scan

## 目的

提供单页扫描模式，仅扫描当前游戏窗口中可见的成就列表页面（含自动滚动到底部），不执行任何 Tab 切换操作。支持多次扫描结果累积合并。

## 需求

### 需求:单页扫描执行
系统必须在单页扫描模式下，仅扫描当前游戏窗口中可见的成就列表页面（含自动滚动到底部），不执行任何一级或二级 Tab 切换操作。

#### 场景:扫描当前可见页面
- **当** 用户在单页扫描模式下点击"开始扫描"
- **那么** 系统必须对当前游戏窗口中可见的成就列表执行滚动扫描（复用 scan_with_scroll 逻辑），直到检测到底部后停止

#### 场景:不执行Tab切换
- **当** 单页扫描模式执行扫描
- **那么** 系统禁止自动点击一级 Tab 图标或二级 Tab 列表，必须保持游戏窗口当前所在的 Tab 页不变

### 需求:单页扫描结果累积
系统必须在单页扫描模式下，将多次扫描的结果累积合并，而非每次清空。

#### 场景:首次单页扫描
- **当** 用户在没有已有扫描结果时执行单页扫描
- **那么** 系统必须将扫描结果直接显示在结果预览表格中

#### 场景:多次单页扫描累积
- **当** 用户已有之前单页扫描的结果，再次执行单页扫描
- **那么** 系统必须将新扫描结果与已有结果合并，基于成就 ID 去重，相同 ID 的条目以最新扫描结果覆盖

#### 场景:全局扫描清空结果
- **当** 用户切换到全局扫描模式并开始扫描
- **那么** 系统必须清空之前所有扫描结果，重新开始全量扫描

### 需求:单页扫描进度显示
系统必须在单页扫描模式下显示适当的进度信息。

#### 场景:显示单页扫描进度
- **当** 单页扫描正在进行中
- **那么** 系统必须显示"单页扫描中"状态文字，以及已识别的成就数量和已完成数量
