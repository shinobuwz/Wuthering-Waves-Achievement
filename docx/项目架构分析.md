# 鸣潮成就管理器 - 项目架构分析

> 版本：2.2.9 | 分析日期：2026-03-22

## 项目简介

鸣潮（Wuthering Waves）游戏成就管理桌面工具。基于 PySide6 构建，支持成就数据爬取、进度管理、统计可视化、多用户档案和数据导入导出。

## 技术栈

| 类别 | 技术 | 版本要求 |
|------|------|----------|
| GUI 框架 | PySide6 | >= 6.4.0 |
| 网络请求 | requests | >= 2.28.0 |
| HTML 解析 | beautifulsoup4 | >= 4.11.0 |
| Excel 处理 | openpyxl | >= 3.0.0 |
| 版本比较 | packaging | >= 21.0 |
| 打包工具 | Nuitka | - |

## 核心架构

### 架构模式

- **MVC**：Config (Model) / Tabs (View) / Manager (Controller)
- **Signal/Slot**：通过 `signal_bus.py` 实现组件间解耦通信
- **Singleton**：全局 `config` 和 `signal_bus` 实例

### 全局信号总线 (`signal_bus.py`)

```
settings_changed        # 设置更新
user_switched           # 用户切换
theme_changed           # 主题切换
category_config_updated # 分类配置变更
crawl_started/finished/progress  # 爬取事件
auth_data_updated       # 认证信息变更
update_available        # 检测到新版本
log_message             # 日志事件
```

## 数据流

### 成就数据流

```
库洛 Wiki API (api.kurobbs.com)
        │
        ▼
   爬取 & 解析 (CrawlTab)
        │
        ▼
  achievement_cache.json (API 响应缓存)
        │
        ▼
  base_achievements.json (成就数据库)
        │
        ▼
   用户进度 (按用户独立存储)
        │
        ▼
   管理页展示 / 筛选 / 编辑
        │
        ▼
   导出 JSON / Excel
```

### 成就数据结构

```json
{
  "绝对编号": "10100001",   // 排序用
  "编号": "10100001",       // 数据持久化用
  "名称": "成就名称",
  "版本": "2.0",
  "第一分类": "索拉漫行",
  "第二分类": "索拉的大地·今州",
  "描述": "成就描述文本",
  "奖励": "星声*10",
  "是否隐藏": "隐藏",
  "获取状态": "已完成",      // 已完成 / 未完成 / 暂不可获取
  "成就组ID": "",           // 互斥成就组
  "互斥成就": []            // 互斥成就列表
}
```

### 编号规则

编号 = `第一分类排序(1位)` + `第二分类后缀(3位)` + `序号(4位)`

示例：`1010` + `0001` → 第1大类、第10小类、第1条成就

## 核心模块详解

### 1. 成就管理页 (`manage_tab.py`)

- 成就表格：点击切换完成状态，长按标记"暂不可获取"
- 多维筛选：文本搜索、版本、一级/二级分类、隐藏状态、获取状态
- 导入导出：JSON 和 Excel 双格式
- 智能重编码：分类变更时自动迁移用户进度数据

### 2. 数据爬取页 (`crawl_tab.py`)

- **数据源**：库洛社区 Wiki API (`POST api.kurobbs.com/wiki/core/catalogue/item/getEntryDetail`)
- **认证**：需要 `devcode` + `token`（存储在 Windows 注册表 QSettings 中）
- **缓存策略**：缓存存在则直接使用，无自动过期机制，需手动"清除缓存"后重新爬取
- **流程**：输入版本号 → 爬取 → 预览 → 确认覆盖到成就库

### 3. 统计图表页 (`statistics_tab.py`)

- 饼图和水平柱状图展示完成率
- 按版本、分类维度统计
- 支持悬浮提示和主题适配

### 4. 配置管理 (`config.py`)

| 存储位置 | 内容 |
|----------|------|
| `resources/config.json` | 用户列表、主题、窗口设置 |
| `resources/base_achievements.json` | 成就数据库 |
| `resources/category_config.json` | 分类映射配置 |
| Windows 注册表 (QSettings) | 认证凭据 (devcode/token) |
| 用户目录 | 每用户独立的成就进度 |

### 5. 更新系统 (`update.py`)

- 通过 GitHub API 检查 `sliencezhang/Wuthering-Waves-Achievement` 仓库最新 Release
- 启动后延迟 3 秒后台检查，不阻塞 UI
- 带缓存机制，避免 API 频率限制
- 备用下载：蓝奏云 (`wwbml.lanzoum.com`)

## 分类体系

### 一级分类（4类）

| 分类 | 说明 |
|------|------|
| 索拉漫行 | 探索相关 |
| 长路留迹 | 剧情/任务相关 |
| 铿锵刃鸣 | 战斗相关 |
| 诸音声轨 | 其他系统 |

### 二级分类

每个一级分类下有多个二级分类，配置在 `category_config.json` 中，支持动态新增和排序调整。

## 构建与发布

### 开发运行

```bash
pip install -r requirements.txt
python main.py
```

### 打包为 exe

```bash
python build.py
```

Nuitka 打包配置：
- `--standalone --onefile`：单文件独立 exe
- `--windows-disable-console`：无控制台窗口
- `--enable-plugin=pyside6`：Qt 支持
- `--lto=yes`：链接时优化
- 排除 numpy/matplotlib/PIL/scipy/pandas 等无用大包

输出：`dist/鸣潮成就管理器{VERSION}.exe`

**注意**：`resources/` 目录需与 exe 同级放置，资源文件外置不打包进 exe。
